# Plan 09-02: Settings Screen + Pool Rebuild

**Phase:** 09-polish  
**Depends on:** 09-01-PLAN.md  
**Requirements:** SETT-01, SETT-02, SETT-03, SETT-04

## Goal

From **discovery**, open a **full-screen Settings** view (gear). User edits **diet** and **cuisine blocklist** using onboarding-like patterns. Preferences save to Zustand immediately. On **Back**, return to discovery and **rebuild the recipe pool** from Supabase using new diet + client-side blocklist filter.

## Tasks

### Task 1: Blocklist filter helper

**New file:** `lib/utils/recipe-filters.ts` (or extend `recipe-urls` — prefer small dedicated module)

```typescript
export function filterRecipesByBlocklist(
  recipes: Recipe[],
  blocklist: string[],
): Recipe[]
```

Exclude any recipe where `cuisine_tags` (or fallback `cuisine`) intersects `blocklist` (case-sensitive match to onboarding chip strings).

**Verify:** Unit-style sanity: empty blocklist returns same array; blocked tag removes recipe.

### Task 2: `rebuildPoolFromPreferences` (or inline in page)

Centralize pool rebuild:

1. `const raw = await fetchRecipes(preferences.diet)`
2. `const filtered = filterRecipesByBlocklist(raw, preferences.blocklist)`
3. `setPool(shuffleArray(filtered))`

Use in **one place** called from:

- Future: optional consolidation with initial greeting fetch (Claude’s discretion — avoid large refactor unless trivial).

**File:** `lib/supabase/recipes.ts` — no schema change required; keep `fetchRecipes` as-is.

**Verify:** After changing diet/blocklist in settings and pressing Back, `session.pool` reflects new data.

### Task 3: `SettingsScreen` component

**New file:** `components/settings/SettingsScreen.tsx`

- Full-screen, scrollable (`min-h-screen`, `max-w-md`, border-x consistent with app).
- **Header:** Back button + title **“Settings”** (or “Preferences”).
- **Section 1 — Diet:** Reuse visual pattern from `DietStep` (three vertical cards, icons) — can import shared constants `DIET_OPTIONS` by extracting to `lib/constants/diets.ts` **or** duplicate minimal markup (Claude’s discretion).
- **Section 2 — Blocklist:** Reuse `CUISINES` + rotated chip pattern from `BlocklistStep` (extract shared `CUISINES` to `lib/constants/cuisines.ts` if not already shared — **DRY** preferred).
- On change: `setDiet` / `setBlocklist` from `useSessionStore` (immediate persist via Zustand).

**No** “Step 1 of 2” chrome.

**Verify:** Toggling updates store (inspect React devtools or localStorage).

### Task 4: Discovery shell — gear + navigation state

**Files:** `components/discovery/FilterBar.tsx` and/or `app/page.tsx`

- Add **settings** button (Material Symbol `settings` or `manage_accounts` — prefer **`settings`**) with **min 44px** touch target.
- Placement: **left** side of top bar or first slot in the filter row (CONTEXT: discoverable; Claude picks — common pattern: **top-left gear**, shuffle/filter stay right).

**`page.tsx`:**

- State: `const [screen, setScreen] = useState<'discovery' | 'settings'>('discovery')`
- When `screen === 'settings'`, render `<SettingsScreen onBack={() => { setScreen('discovery'); void rebuildPool() }} />` instead of discovery main (hide FilterBar / card stack while in settings, or only replace main — **full-screen** settings covers everything).

**`rebuildPool`:** async function that calls `rebuildPoolFromPreferences`, handles loading state if needed (skeleton or spinner — Claude’s discretion).

### Task 5: Optional — `syncSession` after rebuild

After successful pool rebuild from settings back navigation, call `syncSession(sessionId, diet, blocklist, cuisineFilter, mealTypeFilter, ingredientFilter)` with **current** session filter snapshot — aligns analytics with Phase 8.

**Verify:** No duplicate sync spam — call once per back navigation.
