# Plan 09-01: Empty State + Session Filter Reset

**Phase:** 09-polish  
**Depends on:** 09-RESEARCH.md, 09-CONTEXT.md  
**Requirements:** EMPT-01, EMPT-02, EMPT-03, EMPT-04

## Goal

When **filtered** recipes (cuisine + meal filters) are fewer than **5**, show the designed empty state (blob + “Hmm. Nothing here.” + two CTAs). **Reset filters** clears session filters only. **Shuffle anyway** reshuffles the raw pool. **Filter bar** remains available in the empty branch (layout refactor).

## Tasks

### Task 1: Store — `clearSessionFilters`

**File:** `stores/session-store.ts`

Add action:

```typescript
clearSessionFilters: () => void
```

Implementation: set `session.cuisineFilter` to `[]`, `session.mealTypeFilter` to `[]`, `session.ingredientFilter` to `null`, and `session.currentIndex` to `0`, `lastActiveAt` to `Date.now()` (mirror `setCuisineFilter` behavior).

Export on `SessionActions` and implement in the store.

**Verify:** TypeScript compiles; action callable from components.

### Task 2: `EmptyDiscoveryState` component

**New file:** `components/discovery/EmptyDiscoveryState.tsx`

- Props: `onResetFilters`, `onShuffleAnyway`
- Visual: Match **hierarchy** of `stitch_screens/6_empty_state.html` — blob-style illustration (SVG or div composition), headline **“Hmm. Nothing here.”**, two buttons:
  - **Reset filters** → `onResetFilters`
  - **Shuffle anyway** → `onShuffleAnyway`
- Neo-brutalist: hard borders, primary/orange accents, Syne/heading font for headline per app globals.
- **Claude’s discretion:** exact SVG vs simplified shapes; button labels can match mockup (“Reset filters” / “Shuffle anyway”).

**Verify:** Component renders without discovery stack; buttons wired.

### Task 3: Refactor `app/page.tsx` — filtered count + layout

1. Import `filterPool` from `@/stores/session-store`.
2. Subscribe to `cuisineFilter`, `mealTypeFilter` (or derive via `useSessionStore` selectors).
3. Compute:

   ```typescript
   const filteredPool = useMemo(
     () => filterPool(pool, cuisineFilter, mealTypeFilter),
     [pool, cuisineFilter, mealTypeFilter],
   )
   const effectiveCount = filteredPool.length
   ```

4. **Empty condition:** `fetchDone && !fetchError && effectiveCount < 5 && pool.length > 0` **OR** handle `pool.length === 0` after failed/empty fetch separately (keep loading/error branches unchanged).

   **Note:** If **raw** `pool.length === 0` after fetch (API empty), show existing error or a minimal message — not necessarily empty state with shuffle; **Claude’s discretion** to align with EMPT (empty state is for “filters too tight”).

5. When `effectiveCount < 5` and user should see empty UI: render `EmptyDiscoveryState` **instead of** `DiscoveryCardStack`, **but** render **`FilterBar`** in the same outer wrapper as today’s discovery branch so filters stay accessible (move `FilterBar` out of the `else` that only runs when `pool.length >= 5`).

6. **Reset filters** handler: `useSessionStore.getState().clearSessionFilters()`.

7. **Shuffle anyway** handler: `useSessionStore.getState().shufflePool()` + optional `logInteraction(sessionId, 'shuffle')` for consistency with `FilterBar`.

8. Adjust `className` on `motion.main` so full-bleed layout applies when showing empty state + filter bar (match discovery min-h-screen).

**Verify:** With tight filters, user sees empty state + filter bar; reset widens pool; shuffle reorders.
