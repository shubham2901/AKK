**2. Loop logic (stores/session-store.ts):**
- nextCard: (effectiveLen?: number) => wrap at edge. Use len = effectiveLen ?? pool.length. If len === 0, no-op. Else: next = (currentIndex + 1) % len.
- prevCard: (effectiveLen?: number) => wrap at edge. Use len = effectiveLen ?? pool.length. If len === 0, no-op. Else: prev = (currentIndex + len - 1) % len.
- DiscoveryCardStack will pass filteredPool.length as effectiveLen when filters active.

**3. Filter state:**
- Add setCuisineFilter: (f: string[]) => set session.cuisineFilter = f, session.currentIndex = 0
- Add setMealTypeFilter: (f: string[]) => set session.mealTypeFilter = f, session.currentIndex = 0
- Add to SessionActions interface
- Persist cuisineFilter and mealTypeFilter (they are part of session, already in partialize)

**4. Export filterPool helper:**
- `filterPool(pool: Recipe[], cuisineFilter: string[], mealTypeFilter: string[]): Recipe[]`
- Logic: recipe.cuisine_tags?.some(c => cuisineFilter.includes(c)) OR cuisineFilter.length === 0; same for meal_type
- Both filters must match (AND). Empty filter = no constraint for that dimension.
- Place in session-store.ts or a small lib; export for use in DiscoveryCardStack.

Per 06-RESEARCH.md: derive options from pool; empty filter = no constraint.
