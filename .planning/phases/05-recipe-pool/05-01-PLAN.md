Update app/page.tsx to orchestrate the recipe pool fetch and handle all UX states.

**Fetch trigger (per CONTEXT):**
- Only when showing GreetingSplash branch: !setupComplete AND pool.length === 0
- On refresh: Zustand hydrates pool from localStorage → pool.length > 0 → skip fetch
- Use useEffect with deps [setupComplete, pool.length, preferences.diet] — but only run fetch when !setupComplete && pool.length === 0

**Fetch flow:**
- Call fetchRecipes(preferences.diet)
- On success: shuffleArray(recipes) from session-store, then useSessionStore.getState().setPool(shuffled)
- On error: set fetchError state (useState: fetchError: boolean or string | null)

**Track fetch completion:** Use a `fetchDone` boolean (useState, default false) to distinguish "still loading" from "fetch returned 0 results". Set to true in both .then() and .catch() of fetchRecipes.

**Discovery branch states (when setupComplete === true):**
1. **Loading:** !fetchDone && pool.length === 0 && !fetchError → show skeleton (simple placeholder: 2-3 card-shaped divs with animate-pulse, or "Loading recipes...")
2. **Error:** fetchError → show empty state with apology: "Sorry, something went wrong." (per CONTEXT)
3. **Empty pool (<5 recipes):** fetchDone && pool.length < 5 → show placeholder: "No recipes match your filters." (Phase 9 will replace with full empty state UI). This covers 0 recipes too.
4. **Ready:** pool.length >= 5 → show discovery content (keep current placeholder "Aaj Kya Khana Hai? Recipes loading soon..." for now — Phase 6 will add cards)

**Important:** fetchError must survive the greeting→discovery transition. Use useState at page level; set it in the catch of fetchRecipes. When setupComplete becomes true, discovery branch reads fetchError.

**Skip fetch on return visit:** If pool.length > 0 when component mounts (e.g. after refresh), do NOT call fetchRecipes. Zustand hydration restores pool.
